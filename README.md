# card_game_demo
Карточный рогалик
Техническое задание на разработку прототипа компьютерной игры "Карточный рогалик"
1. Название проекта: "Dungeon Deck" (или любое другое на ваш выбор)

2. Цель проекта:
Разработать упрощенный прототип карточной игры в жанре roguelike, демонстрирующий ключевые механики жанра: процедурную генерацию, перманентную смерть, постепенное развитие персонажа и стратегическое использование карт.

3. Описание игры:
Игрок проходит через серию последовательных локаций (комнат), встречая случайных противников и события. Бой и взаимодействия решаются с помощью карт, составляющих колоду игрока. После боя игрок получает новые карты и улучшения, чтобы усилить свою колоду для последующих, более сложных сражений.

4. Функциональные требования:

4.1. Базовый игровой цикл:

Начало игры: Создание персонажа (выбор из нескольких предустановленных классов, например, Воин, Маг, Плут) с уникальными стартовыми колодами.

Карта путешествия: Отображение последовательности комнат (линейная или с выбором пути). Типы комнат:

Бой: Сражение с противником.

Событие: Случайное событие с выбором, которое может дать награду или наказание.

Отдых: Восстановление здоровья, улучшение карты или удаление карты из колоды.

Сокровище: Получение новой случайной карты или реликвии (пассивного усиления).

Боевая система (покомандная):

Ходы делятся на ходы игрока и ходы противника.

В начале боя игрок тянет руку (напр., 5 карт) из своей колоды.

Игроки и враги имеют Здоровье (HP) и Броню (Armor). Броня блокирует урон и сгорает в конце хода.

Карты требуют для разыгрывания Маны (Mana), которая восполняется в начале каждого хода игрока (напр., до 3/5/7 единиц).

Типы карт:

Атака: Наносит урон врагу.

Защита: Дает броню.

Навык: Накладывает эффекты (например, Слабость, Уязвимость), лечит, позволяет вытянуть дополнительные карты.

После разыгрывания карт игрок завершает ход. Затем противник совершает свои действия (атакует, защищается, накладывает эффекты) по заранее определенному скрипту.

Победа в бою: получение награды (золото, выбор одной из трех случайных карт, реликвия).

Колода и прогресс:

Возможность просмотра всей своей колоды между боями.

Возможность добавлять новые карты в колоду и удалять существующие в специальных точках (например, в комнатах отдыха).

Система "прокачки" через реликвии, которые дают постоянные пассивные бонусы (например, "В начале боя получай 1 дополнительную ману").

Конец игры:

Проигрыш: Персонаж умирает (HP опускается до 0). Запускается новая игра.

Победа: Прохождение финального босса.

4.2. Технические требования:

Язык программирования: Python (рекомендуется для простоты и наглядности).

Графический интерфейс: Консольное приложение (CLI). Интерфейс должен быть интуитивно понятным и отображать:

Текущее состояние игрока (HP, Мана, Броня, Реликвии).

Состояние противника.

Карты в руке.

Лог событий (какие карты были разыграны, какой урон нанесен и т.д.).

Архитектура: Четкое разделение на модули/классы в парадигме ООП.

5. Нефункциональные требования:

Производительность: Отсутствие "зависаний" и лагов во время игрового процесса.

Надежность: Программа не должна завершаться с ошибкой (крашиться) при стандартных действиях пользователя.

Читаемость кода: Логичная структура проекта, понятные названия переменных и функций, комментарии к сложным участкам кода.

6. Структура проекта (предлагаемые классы и модули):

main.py - Главный файл, управляющий игровым циклом.

game.py - Класс Game, управляющий состоянием игры (карта, текущая комната, игрок).

player.py - Класс Player (HP, мана, броня, колода, рука, реликвии).

entities.py - Классы Entity (базовый), Enemy (наследник Entity с AI-поведением).

cards.py - Базовый класс Card и его наследники (AttackCard, DefenseCard, SkillCard).

relics.py - Класс Relic для пассивных усилений.

map.py - Класс Map для генерации и управления маршрутом комнат.

room.py - Класс Room для описания различных типов комнат.
